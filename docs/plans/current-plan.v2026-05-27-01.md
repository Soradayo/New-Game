# ターン速度・高速進行・時間解像度プロファイル計画

## Summary
- 直近では一生完走の重さを解消するため、標準のターン幅を再設計し、`重要な出来事まで` の自動進行を追加する。
- ただし時間解像度は将来的に Civilization のゲーム速度設計のような「速度プロファイル」として扱える構造にする。
- MVPでは `standard` 速度のみを実装し、将来 `fast`, `slow`, `chronicle` のような速度を追加できる余地を残す。

## Key Changes
- 標準ターン幅:
  - childhood: 12か月/ターン
  - youth: 6か月/ターン
  - youngAdulthood: 3か月/ターン
  - adulthood: 4か月/ターン
  - oldAge: 12か月/ターン
  - 開始6歳、終端80歳で約180ターン。
- 時間解像度プロファイル:
  - `TimeScaleProfile` の概念を設ける。
  - MVPでは `standard` のみ使用する。
  - 将来、短時間周回用の `fast`、長期追跡用の `slow`、詳細記録用の `chronicle` を追加できるようにする。
  - イベント発生率、転機保証年齢、要約ログ密度は将来的にプロファイル影響を受ける設計にする。
- 高速進行:
  - 通常UIに `重要な出来事まで` ボタンを追加する。
  - 停止条件は、転機、`isMajor: true` イベント、80歳到達、最大120ターン到達。
  - 飛ばした通常ログは記録文体の期間要約に畳む。
- 重大イベント:
  - `EventDefinition` に `isMajor?: boolean` を追加する。
  - Mod schemaとテンプレートコメントにも説明を追加する。
- 開発補助:
  - dev環境だけ、年齢ジャンプ、転機強制、80歳到達確認の開発パネルを出す。

## Test Plan
- 新しい `standard` ターン幅で各フェーズが進む。
- 6歳から80歳までが約180ターンで到達する。
- `重要な出来事まで` が転機または重大イベントで止まる。
- 通常イベントは要約ログに畳まれる。
- `isMajor` がschemaとModテンプレートで扱える。
- 将来の速度プロファイル追加を想定し、ターン幅が1か所の設定から参照される。

## Assumptions
- 今回は速度選択UIまでは作らない。
- MVPでは `standard` のみ。
- 速度プロファイルの本格実装は今後の計画に回す。
- ただし今回の実装では、将来プロファイル化しやすい形でターン幅を定義する。
