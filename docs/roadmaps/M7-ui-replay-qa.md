# M7: UI完成・リプレイ・QA

## Summary

完成版1.0として触れる状態に仕上げる。白黒基調の機能美、履歴閲覧性、リプレイ性、レスポンシブ、Modありプレイの確認を重視する。

## Key Changes

- UIは機能美重視で磨く。
  - 白黒基調。
  - 読みやすいログ。
  - 明確な選択状態。
  - レスポンシブ。
  - 履歴閲覧性。
- 履歴タブにフィルタを追加する。
  - 重要ログ。
  - 転機ログ。
  - NPCログ。
  - 世界ログ。
- リプレイでは、要約ログと実ログを切り替えて人生を読み返せるようにする。
- save/export/import、Mod読込、言語切替、完走確認を最終QA対象にする。

## Completion Criteria

- 通常プレイで一生を完走できる。
- 高速進行で老年期まで到達できる。
- 転機、NPC関与、世界変化が履歴として読み返せる。
- Modありでも主要操作が破綻しない。
- モバイル幅でも主要操作が成立する。

## Test Plan

- Browser smoke:
  - 通常ターン。
  - 重要イベントまで進行。
  - 転機選択。
  - NPC関与。
  - 履歴閲覧。
  - 言語切替。
  - Mod読込。
- 完走QA:
  - 通常プレイ。
  - 高速進行。
  - Modあり。
  - 言語切替あり。
- Build/Test:
  - `npm test`
  - `npm run build`

## Assumptions

- 演出より可読性と操作性を優先する。
- 画像やキャラクターグラフィックは1.0範囲外。
- 最終QAでは、テストコードだけでなくブラウザ上の手動確認も行う。
